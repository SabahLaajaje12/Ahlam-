\# Phase 1: Concept and Visual Prototype



\*\*Timeline:\*\* Project Start  

\*\*Status:\*\* ✅ Complete



\## The Vision



Ahlamé is a women-only beauty booking marketplace—not a single salon website, not an e-commerce beauty store. The core concept is a multi-provider marketplace where customers can discover, compare, and book beauty services across multiple providers.



\## Early Decisions



| Decision | Rationale |

|----------|-----------|

| Women-only positioning | Clear target audience, safer space branding |

| Multi-provider from day one | Avoid architectural lock-in to single-salon model |

| FastAPI + Jinja | Python-first, server-rendered for simplicity and SEO |

| SQLite for development | Zero-config, fast iteration |



\## What the Prototype Included



\- Homepage with marketplace feel

\- Provider listing page (grid of available providers)

\- Provider detail page (services, reviews, booking CTA)

\- Booking page skeleton

\- Admin/provider preview pages



\## What Was Placeholder



\- Demo providers hardcoded in HTML

\- Static reviews

\- Non-functional booking buttons

\- No database persistence



\## Lesson Learned



A visual prototype helps align the product direction, but jumping from prototype to working MVP exposes every hidden assumption.
